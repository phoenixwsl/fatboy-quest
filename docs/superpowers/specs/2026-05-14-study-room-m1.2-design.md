## 背景

书房 M1 + M1.1 已完成基础布局（1600×1000 画布 + 等比缩放 + 墙/柜/桌/椅/绿植/画）。用户在 iPad 上测试发现三个问题：

1. iPad 上只显示约 2/3，四周有黑边
2. 绿植太小、位置不自然
3. 奖杯是纯装饰，缺少交互

---

## 一、iPad 布局修复

### 问题根因

`useStageScale()` 用 `Math.min(vw/1600, vh/1000)` 缩放。iPad 横屏 ~1024×768 时：
- sX = 1024/1600 = 0.64
- sY = 768/1000 = 0.768
- scale = 0.64 → 实际画面 1024×640，高度空出 128px

`.study-page` 背景是纯深棕 `#2A1F18`，空出区域显示为黑边。

### 方案：背景延伸

保持 `Math.min` 等比缩放不变（确保内容不被裁切），将 `.study-page` 的背景从纯色改为与 stage 内部一致的墙面 + 地板渐变：

```css
.study-page {
  /* 保留原有 flex 居中 */
  background:
    linear-gradient(
      to bottom,
      var(--study-wall-top) 0%,
      var(--study-wall-bottom) 69%,   /* FLOOR_LINE_Y / STAGE_H */
      var(--study-floor) 69%,
      var(--study-floor) 100%
    );
}
```

**效果**：空出的区域自然显示为"墙壁和地板的延伸"，无黑边。

- `src/components/home/study-room.css` — `.study-page` 背景

## 三、奖杯点击浮层

### 交互流程

```
点击左柜奖杯 SVG
  → 浮层 slideUp（spring 弹性）
    → 网格展示所有已解锁成就（3-4 列，可纵向滚动）
    → 最新获取的在最上面
    → 点击单个奖杯 → 展开详情（emoji 放大 + 标题 + 描述 + 获取时间）
    → 再次点击或点其他奖杯 → 收起
  → 点击遮罩或 ✕ → 浮层 fadeOut + slideDown
```

### 数据来源（无新 schema）

```typescript
// 已有
db.badges.toArray()  // → Badge[] { id, unlockedAt }
ACHIEVEMENTS         // → Achievement[] { id, title, description, emoji, type }

// 合并
const trophies = badges
  .map(b => ({ ...b, ...ACHIEVEMENTS.find(a => a.id === b.id) }))
  .sort((a, b) => b.unlockedAt - a.unlockedAt);
```

### 动效规格

| 动作 | 动画 | 时长 |
|------|------|------|
| 浮层进入 | 遮罩 fadeIn + 内容 slideUp (spring) | ~0.3s |
| 浮层关闭 | 遮罩 fadeOut + 内容 slideDown | ~0.2s |
| 奖杯展开 | framer-motion `layout` 平滑放大 | ~0.25s |
| 奖杯收起 | framer-motion `layout` 平滑缩回 | ~0.2s |
| 奖杯点击 | whileTap scale(0.95) | 即时 |

### 空状态

没有解锁任何成就时显示：
> 🌟 继续加油，第一个奖杯在等你！

措辞遵循 ADHD 友好原则 — 鼓励而非"暂无"。

### ADHD 友好检查

- ✅ 网格布局整齐，视觉负担低（工作记忆友好）
- ✅ 点击才展开详情（信息按需加载）
- ✅ 最新在最上面（即时成就感 → 多巴胺正反馈）
- ✅ 自然滚动，不分页（零认知负担）
- ✅ 不离开书房（任务切换成本低）
- ✅ 空状态温柔鼓励（情绪友好）
- ✅ 关闭简单（点遮罩/✕，不需要找按钮）
