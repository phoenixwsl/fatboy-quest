# 肥仔的书房 · 完整设计规格

> **目标读者**：Frank（Product Owner）+ Claude Code（执行者）
> **状态**：v1.0 设计定稿，待开发
> **预计工期**：4-5 周业余时间

---

## 0. 这是什么

**肥仔的书房**是肥仔大闯关的"第二战场"——

- 第一战场（现有）：日常做作业 → 拿积分 → 升段位 (驱动当下行为)
- **第二战场（新增）：积分 → 收藏 → 陈列 → 看见自己积累的力量 (驱动长期留存)**

设计灵感：博物馆 + 工作室 + 日式收藏屋。
情感目标：让儿子打开 app 后**想多看几眼**，离开后**想着自己的柜子又添了什么**。

**不做什么**：
- ❌ 不做装修（家具搬动、自由摆放）
- ❌ 不做心愿单（不打通现实购买）
- ❌ 不做任务委托（不引入 NPC 系统）
- ❌ 不做多个房间（只一个书房）

## 1. 整体布局

### 视角
**正面视角**——摄像机在书桌前方对着肥仔，肥仔正脸朝向观众。

构图原则：信息密度高但有层次。从远到近 5 层：

```
图层 1 · 墙面（米黄/木质护墙板）
图层 2 · 三幅画（墙上）
图层 3 · 三个收藏柜（紧贴墙边）
图层 4 · 椅背（深色现代办公椅）
图层 5 · 肥仔（露上半身）
图层 6 · 书桌 + 摆件（前景）
```

### 画布比例
- 桌面/移动：建议 16:10 横屏布局
- 移动竖屏：垂直堆叠，三柜变为可横滑

### 屏幕分区

| 区域 | 高度占比 | 内容 |
|---|---|---|
| 顶部导航 | 7% | [← 退出] 标题 [⚙ 设置] |
| 三幅画 | 14% | 装饰画（主题对应下方柜子） |
| 三个柜子 | 41% | 奖杯柜（左）/ 乐高柜（中大）/ 玩具柜（右） |
| 肥仔区 | 18% | 椅背 + 肥仔上半身 + 桌后摆件 |
| 书桌 + 摆件 | 13% | 桌面、笔记本、笔筒、地球仪等 |
| 底部 bar | 7% | 积分 + 「装饰商店」按钮 |

### 横向比例（三柜）
中柜略宽：**左:中:右 = 1 : 1.3 : 1**
- 中柜放乐高（实物照片需要更大显示空间）

## 2. 视觉 DNA

| 护墙板线 | `--study-wainscot-line` | `#8C6446` |
| 木质暗 | `--study-wood-dark` | `#593C23` |
| 木质中 | `--study-wood` | `#825A37` |
| 木质亮 | `--study-wood-light` | `#B4875A` |
| 桌面 | `--study-desk-top` | `#D7AF7D` |
| 桌面亮 | `--study-desk-top-light` | `#EBC89B` |

| 椅子缝线 | `--study-chair-stitch` | `#4B321E` |
| 暖光晕 | `--study-warm-glow` | `rgba(255,200,130,.4)` |

**关键决策**：书房**全天都是暖橙调**，不跟随外部白天/夜晚切换。
这是"室内永远温暖"的设定，让儿子任何时候打开都有安心感。

### 字体
- 标题：阿里巴巴普惠体 Bold
- 数字：Fredoka + tabular-nums
- 标牌字体（柜子上方铜牌）：阿里巴巴普惠体 Medium 16pt

### 灯光
- 整体：暖色 ambient（从顶部洒下）
- 三柜内部：各有一束**聚光灯**（warm 60% opacity 椭圆光晕）
- 台灯：周围有局部光晕
- 肥仔：脸部有微弱补光

### 质感细节（不能省）
- 柜子玻璃门反光（两道斜向白光，opacity 25%）
- 柜子内部"小聚光灯"（顶部圆形渐变）
- 桌面木纹理（细横条纹）
- 护墙板的板间分隔线
- 椅背的车线（vertical stitching）—— 这是"逼格"关键

### 3.2 乐高柜（中，最大）

**总数**：12 个乐高陈列位
**橱窗显示**：4 排 × 2 列 = 8 个
**详情页**：完整 4 × 3 = 12 个网格

#### 关键设计：实物图片

每个乐高位置展示**真实产品照片**（不是 SVG 绘制）：
- 透明背景的产品图
- 加暖光叠加（5% 暖白 overlay）
- 圆角 8px 裁剪
- 暖橙阴影投影

这是产品最有"质感"的部分——**虚拟世界里展示真实物品**。

#### 商品库（Frank + 爱人精选）

建议先选 **12 款**精选乐高，分三档：

| 档 | 价格区间 | 积分价 | 数量 |
|---|---|---|---|
| 入门 | 200 元以下 | 20-30 ⭐ | 4 款 |
| 进阶 | 200-600 元 | 30-60 ⭐ | 4 款 |
| 高级 | 600-1500 元 | 60-150 ⭐ | 3 款 |
| 旗舰 | 1500+ 元 | 150+ ⭐ | 1 款（如法拉利 SF90） |

**积分换算公式**：现实价格 ÷ 10 = 积分价格。让虚拟"购买"有重量。

## 4. 三幅墙画

**位置**：每幅画挂在对应柜子正上方
**尺寸**：宽度约为柜宽的 70%
**风格**：油画质感、暖色调、与下方柜子主题呼应

| 位置 | 主题 | 内容 |
|---|---|---|
| 左（奖杯柜上方） | 荣耀 | 金桂冠 + 星星（致敬奖牌精神） |
| 中（乐高柜上方） | 想象 | 城堡剪影 + 日落（建筑梦想） |
| 右（玩具柜上方） | 探索 | 星空 + 火箭 + 月亮（冒险精神） |

**重要约定**：
- 画**不是商品**，不可购买替换
- 画是这个书房世界观的"装饰"，固定 3 幅
- 后续 v2 可考虑加可购买的画，但 v1 不做

## 5. 书桌摆件

**摆件总数**：8-12 种（每种可解锁一个）
**桌面布局**：固定槽位，每个摆件出现在指定位置

### 摆件槽位（从左到右）

| 位置 | 槽位 | 推荐摆件 |
|---|---|---|
| 桌左 | 台灯 | 复古绿罩台灯 / 北欧木质 / 未来 LED |
| 中左 | 笔筒 | 蓝色笔筒（含彩色铅笔） |
| 中央前 | 笔记本（默认有） | 打开的作业本 + 铅笔 |
| 中央后 | 时钟（默认有） | 圆形台钟 |
| 中右 | 地球仪 | 经典地球仪 / 电子地球仪 |
| 桌右 | 相框 | 全家福 / 励志语录框 |
| 角落小件 | 装饰 | 小植物 / 宇航员手办 / 机器人 |

### 摆件设计原则
- **默认有 2 个**：笔记本和时钟（让桌面不空）
- **其余 6-10 个**通过积分购买
- 每个槽位**只有一个摆件可见**（同槽位的可切换显示）
- 已购买未展示的可在"摆件库"里切换

### 数据模型

  "id": "lamp_retro_green",
  "category": "desk_item",
  "slot": "left",
  "name": "复古绿罩台灯",
  "price_points": 50,
  "image_svg": "lamp_retro_green.svg",
  "description": "图书馆同款",
  "default_owned": false

## 6. 肥仔本人

### 位置
画面中央，露出上半身（脖子以上 + 肩膀），其余被书桌遮挡。

### 状态
**永远是 default 状态**——肥仔在这里是"主人"，不是"工作中"，所以是放松的笑脸。

> 不要根据外部产品状态切换 focused/tense/etc.
> 这是"温馨治愈"的核心——书房是避风港。

### 角色联动
肥仔的角色（赛车手/宇航员/海盗/etc.）**跟随主产品的角色选择**：
- 用户在外部选了赛车手 → 书房里肥仔也是赛车手
- 椅背风格相应变化（见下）

### 椅背设计（重要的人格化元素）

| 角色 | 椅背风格 |
|---|---|
| default | 棕色皮革办公椅（默认） |
| racer | 红色赛车椅靠头 + 安全带肩部凸起 |
| astronaut | 银色飞船驾驶座 + 多按钮装饰 |
| pirate | 木质船长椅 + 黄铜钉装饰 |
| ninja | 深黑木椅 + 暗红绑带 |
| mario | 红色蘑菇靠垫 |
| knight | 铁质王座 + 红色软垫 |
| wizard | 深紫天鹅绒椅 + 金边 |

**8 种椅背的实现成本**：每个 SVG 约 1-2 KB，总共约 10 KB 资产。
**优先级**：v1 先做 default 椅背，其他 7 个 v1.1 再补。

## 7. 交互行为

### 7.1 进入书房

**入口位置**：
- 主产品首页底部（不抢戏的小卡片，红点提示新奖杯）
- 家长面板内（独立入口）

**进入动画**：
- 0.0s：当前页面淡出
- 0.5s：黑屏（短暂转场）
- 0.5-1.0s：书房从黑屏淡入 + "推开门"音效
- 1.0s：完全显示，肥仔挥手/眨眼

### 7.2 退出书房

点击左上角 [← 退出]：
- 0.2s 内淡出 + "关门"音效
- 返回到进入书房前的页面

### 7.3 点击三柜（最重要交互）

**点击柜子**（点击任意陈列品或柜子本体）：
- 0.3s 内柜子放大铺满屏幕（spring 动画）
- 进入**全屏陈列页**

**全屏陈列页布局**：

**陈列页交互**：

- 锁定物品：点击 → "完成 XXX 来解锁这个奖杯" 提示（不显示具体内容，神秘感）

### 7.4 点击单个物品 → 详情卡

如 §3.1 所示，详情卡是 modal，背景模糊。

### 7.5 点击装饰商店

底部右下角的黄色 2.5D 按钮：
- 进入商店页（独立路由）
- 商店内分 3 个 tab：乐高 / 玩具 / 摆件
- 已拥有的灰显 + "✓ 已收藏"
- 积分够的高亮 + "立即收藏"

## 8. 数据模型

### 8.1 用户的收藏数据

```typescript
interface UserCollection {
  user_id: string;

  // 奖杯
  trophies: {
    id: string;
    unlocked_at: Date;        // 解锁时间
    rarity: 'common' | 'rare' | 'epic' | 'legendary';
  }[];

  // 乐高
  legos_owned: {
    id: string;
    purchased_at: Date;        // 购买时间
    points_paid: number;
    parent_note?: string;      // 家长留言（可选）
  }[];

  // 玩具
  toys_owned: {
    id: string;
    purchased_at: Date;
    points_paid: number;
  }[];

  // 摆件
  desk_items_owned: string[];  // ID 列表
  desk_items_active: {         // 当前展示的（每槽位一个）
    slot: 'left' | 'middle_left' | 'middle' | 'middle_right' | 'right' | 'corner';
    item_id: string;
  }[];
}
```

### 8.2 商品目录（admin 维护）

```typescript
interface ItemCatalog {
  trophies: Trophy[];          // 35 个固定
  legos: LegoItem[];           // 12 个，Frank + 爱人精选
  toys: ToyItem[];             // 12 个，同上
  desk_items: DeskItem[];      // 8-12 个 SVG
  paintings: Painting[];       // 3 个固定（不可购买）
}
```

### 8.3 解锁逻辑

```typescript
// 每次完成任务后调用
function checkAndUnlockTrophies(user: User, event: TaskCompletedEvent) {
  for (const trophy of ALL_TROPHIES) {
    if (user.trophies.find(t => t.id === trophy.id)) continue;
    if (trophy.checkUnlock(user, event)) {
      user.unlockTrophy(trophy.id);
      showUnlockAnimation(trophy);
    }
  }
}
```

### 8.4 隐藏奖杯如何"不可见"

数据上仍然存在，只是 UI 上不展示触发条件：
- 锁定时显示"???"而非"完成 XXX"
- 解锁前不能在陈列页搜索
- 解锁瞬间播放特殊动画（金光爆发）
