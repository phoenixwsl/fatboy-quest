## B. 4 大共性 — 它们做对了什么

1. **单一强主色 + 大面积中性底**：主色不超过 1-2 个，60-70% 视觉是奶白 / 米白容器。视觉焦点单一。
2. **状态色独立成系统**：成功(绿)、错误(红)、警示(橙)、信息(蓝)永远独立。Duolingo 体现最彻底（streak 橙 / heart 红 / gem 蓝 / xp 黄）。
3. **大圆角 + 可触摸形状**：最小 12px，常态 20-32px；按钮带 3-4px 下沉阴影营造可按压感。儿童产品 0% 出现尖角。
4. **饱和度不堆叠**：一屏内只让一个区域满饱和，其余降饱和或转中性，靠"色彩稀缺性"凸显 CTA。
5. **吉祥物 / 装饰元素做记忆锚点**：Duo 猫头鹰、ACNH 树叶贴纸、Kirby 星星和云朵 — 主题靠"形状装饰"而不仅靠颜色。
6. **字体圆头化**：所有标杆都用圆头 sans 或定制圆体。几何方正体在儿童产品中已淘汰。

## D. 大师与色彩理论（UI 可落地版）

### James Gurney《Color and Light》

1. **阴影色温反相** — 亮部暖色（米黄 paper），阴影/低层卡片就用**冷灰偏蓝**。直接对应"卡片 hover/pressed 状态"色。反之亦然。
2. **灰色用补色调和而非黑白** — `--ink-muted` 不要 `#888`，用"主色补色 + 主色"按 7:3 混合，灰色就有主题归属感。
3. **统一光源 / 统一色调（color unity）** — 一个主题里所有色都"加一点"主色（cozy 里连灰色也带 5% 暖橙），整屏像被同一盏灯照过。
4. **饱和度梯度而非明度梯度** — 区分层级靠"主色 80% / 50% / 20% 饱和"，比单纯调亮调暗高级。
5. **补色阴影 = 视觉张力** — CTA 按钮下方的阴影用主色补色（绿按钮配紫红阴影），比黑色阴影"活"。

### Joseph Albers《Interaction of Color》

1. **同一颜色在不同底上"看起来不同"** — 深色主题下 `--ink: #F0F4FF` 看着 OK，放到浅蓝 paper 上瞬间糊掉，必须按主题分别校准文字色，不要复用。
2. **同时对比（simultaneous contrast）** — 灰色放进彩色环境会自动染上补色；深色主题里的"中性灰"必须主动偏向冷蓝或暖棕，否则视觉自动染色。
3. **色彩相对性 → 系统化语义** — 同一个红，在白底是"警告"，在黑底是"主色高亮"。`--danger` 在不同主题要分别定义，不能复用一个 hex。
4. **少即多** — 3 色搭配比 7 色搭配更有秩序感。每个主题"敢用的色"控制在 5 个以内。

### Johannes Itten 七大对比

1. **明度对比（light-dark）是第一性** — WCAG 4.5:1 不是终点，正文 7:1 才有"质感"。深色主题里 ink 不要 `#F0F4FF` 这种偏冷的近白，改 `#FFFFFF` 或 `#FAFBFF`，对比 +20%。
2. **冷暖对比定义主题情绪** — cozy = 全暖，mecha = 全冷，starry = 冷底 + 暖星点。**每个主题选一个温度方向且只选一个**。
3. **互补对比用于 CTA** — 按钮主色取主题色对角互补色（cozy 暖橙 → CTA 用青蓝），点击率天然高。
4. **比例对比（extension）** — Itten 比例：黄:紫 = 3:9，橙:蓝 = 4:8，红:绿 = 6:6。强色面积越小越提神 — CTA 永远小面积。
5. **饱和度对比** — 一屏内只允许一处"纯饱和"，其他统统降到 60% 以下，否则"幼稚"而非"童趣"。
6. **同时对比（与 Albers 重合）** — 见上。
7. **质地对比（quality contrast）** — Itten 第七对比少有人提：磨砂 vs 光泽 vs 透明。深色主题用 glow 营造"光泽"，浅色主题用 paper 营造"磨砂"。

## E. Apple Design Award 加分案例

### Procreate（2013, 2018, multi-year）
- 色板完全用户自定义；启示：**主题色做成 token，避免硬编码**。
- 工具栏深灰 `#1F1F1F` + 白色画布 + 用户彩色 — "克制中性 + 用户自由色"是创作工具的范式。

### Monument Valley 2（2017）
- 关键色板：`#218380 / #83C5BE / #FDE9E2 / #E29578 / #CC1F5E`
- **低饱和绿松 + 高饱和珊瑚红**是儿童产品的高级感秘诀 — 大面积低饱和打底，小面积高饱和强调。

### Alto's Odyssey（2018）
- 渐变：`#FFA4B9 → #513869 → #271538`
- 启示：**背景用渐变而非纯色**，立刻"有质感"。 整屏渐变可随时间推进缓慢漂移。

### Sky: Children of the Light（2020）
- thatgamecompany 出品，色彩叙事一流。
- 启示：**冷底唯一暖色**作为情感锚点。整个云海是冷紫蓝，所有"灯火"是暖橙 — 唯一暖色 = 视觉锚点。

### Linear（2020 redesign）
- 用 **LCH 色彩空间**生成主题，保证不同主题"知觉明度"一致。
- 启示：**强烈建议 fatboy-quest 用 OKLCH**。

### Carrot Weather（2017）
- 语义色（蓝=雨、红=热、灰=阴）从不和品牌色混用。
- 启示：**状态色独立的范本**，与品牌色物理隔离。

### Things 3（2017）
- 极致克制：白 + 蓝 + 黄三色全产品。
- 启示：**3 色 < 7 色**。少色也能高级。

### Bear（2017）
- OLED 黑 `#000000` + 单一强调色（用户选）。
- 启示：**OLED 黑省电且高端**。但不适合儿童 — 太冷感。

### Tot（2020）
- 7 个固定 dot 色作为视觉系统全部。
- 启示：**色板可以小到 7 个**，全产品复用。

## F. 反例（什么不要学）

- **Bootstrap 默认配色** — `#007bff / #6c757d / #28a745 / #dc3545 / #ffc107`。色相过分饱和、缺乏温度统一、状态色互相打架。新项目一上来就是这套，等于宣告"我没想过设计"。
- **Material Design 早期 indigo+pink** — 互补对比是对的，但饱和度都拉满，长期看刺眼。Material 后来转向 Material You 是对的方向。
- **所有"渐变到死"的设计** — iOS 早期 instagram、podcast 那种紫粉蓝渐变现在看就 dated。渐变要克制，背景大渐变可以，每个按钮渐变就 over 了。
