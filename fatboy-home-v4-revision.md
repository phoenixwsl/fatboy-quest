## C1 · 中央画替换

### 1.1 设计意图

之前 v3 中柜上方的画是"夕阳城堡剪影"——泛主题装饰。
v4 把它换成**用户的偶像海报**（这一版是科比投篮）。

**为什么这是关键改动**：
- 这不是装饰，是孩子的**身份宣告**
- 每个孩子卧室都有偶像海报——这是他童年记忆
- 比城堡/星空更有个人锚点
- 看到这张画，孩子想起"为什么要努力"

### 1.2 资产规格

**用户提供的图片**：
- 位置：`/src/assets/home/paintings/center_hero.jpg`
- 推荐尺寸：800 × 500 px（4:2.5 比例匹配画框内部）
- 格式：JPG（带轻度油画/水彩滤镜）或 PNG
- Frank 已提供初始图：科比 SF 投篮夕阳油画

**Frank 后续维护建议**：
- 保留一个 `/center_hero.jpg` 作为默认偶像图
- 后续 v4.1 可做"上传自定义偶像图"功能（不在 v4 范围）

### 1.3 渲染规则

```tsx
// PaintingCenter.tsx
<div className="painting-frame painting-center">
  <div className="painting-frame-outer" />
  <div className="painting-frame-middle" />
  <div className="painting-frame-inner" />
  <div className="painting-mat">
    <img
      src="/assets/home/paintings/center_hero.jpg"
      alt="偶像海报"
      className="painting-image"
    />
  </div>
</div>
```

```css
.painting-image {
  width: 100%;
  height: 100%;
  object-fit: cover;       /* 充满画框，可能裁剪 */
  border-radius: 2px;
}
```

### 1.4 画框样式不变

继承 v3 的画框设计（木质 3 层框 + 衬纸 + 画轨挂绳）。
只替换内部内容。

### 1.5 三幅画的最终定位

| 位置 | 主题 | 内容 |
|---|---|---|
| 左（奖杯柜上方） | 荣耀 | 金桂冠 + 星星（SVG 绘制，不变） |
| **中（乐高柜上方）** | **个人偶像** | **用户提供的偶像图片** |
| 右（玩具柜上方） | 探索 | 星空 + 火箭 + 月亮（SVG 绘制，不变） |

## C2 · 桌面颜色加深

### 2.1 问题

v3 桌面颜色 `#D7AF7D`（浅木）和地板 `#BE9873`-`#A07854`（浅木渐变）**色相过近**，桌子像融在地板里。

### 2.2 改动

| 部位 | v3 色值 | v4 色值 | 变化 |
|---|---|---|---|
| 桌面顶部 | `#E1C396` | `#8B5A2B` | 浅木 → 深胡桃 |
| 桌面侧面 | `#C8A578` | `#704721` | 中木 → 深胡桃 |
| 桌前边缘 | `#966E46` | `#3E2614` | 深木 → 深棕黑 |
| 桌前面板 | `#AF8C5F` | `#5C3920` | 中木 → 深棕 |
| 桌面木纹线 | `#BE9B69` | `#A07050` | 浅 → 中深 |

### 2.3 设计原理

家装的"前景对比"原则：**前景物体应比中后景深一档**。
这样肥仔（暖黄）站在深桌面前，对比强烈，主角凸显。

### 2.4 实施

更新 `study-room-tokens.css`：

```css
:root {
  /* 修改这 5 个 token */
  --study-desk-top:       #8B5A2B;
  --study-desk-top-light: #A07050;
  --study-desk-edge:      #3E2614;
  --study-desk-front:     #5C3920;
  --study-desk-grain:     #A07050;
}
```

桌前面板高度从 70px **压缩到 36px**，让椅子下半部分露出一些（不要全遮挡）。

## C3 · 椅子放大 + 肥仔上移

### 3.1 问题

v3 试图突出肥仔（椅子缩到 240，肥仔放大到 260），但带来反作用：
- 肥仔头被椅背圆角"夹住"
- 肥仔太靠下，几乎被桌面压扁
- 椅子太小看起来像玩具

### 3.2 改动

回归 v2 的人物比例，但**位置上移**：

| 项目 | v3 | v4 |
|---|---|---|
| 椅子宽度 | 240 | **360**（v2 同款，稳重） |
| 椅子高度 | 130 | **150** |
| 肥仔尺寸 | 260 | **260**（保持） |
| 肥仔 y 起点 | 690 | **600**（上移 90px） |
| 露出部分 | 头 + 一点脖子 | **头 + 脸 + 肩 + 一点身体** |

**关键**：肥仔的脸不应该被桌面切到下巴位置——应该看到**完整的脸 + 肩**。

### 3.3 椅子设计微调

椅背顶部从"大圆角"改为"较平直" + 微弧顶（让肥仔头不被夹住）：

```css
.chair-back {
  /* v3 */
  border-radius: 36px;

  /* v4: 顶部更平直 */
  border-top-left-radius: 24px;
  border-top-right-radius: 24px;
  border-bottom-left-radius: 36px;
  border-bottom-right-radius: 36px;
}
```

### 3.4 椅子位置

椅子顶部 y 从 750 调到 **720**（与肥仔肩部接触，不再"夹"肥仔）。

## C4 · 桌后地面绿植

### 4.1 设计意图

v3 留下的问题：柜子和椅子之间有一段地板空白（柜底 685 → 椅顶 720），视觉空洞。
绿植填补这段地板，同时引入"可购买换新"的玩法。

### 4.2 位置规格

```
柜子底部 y = 685
↓ 35px 留白
绿植根部 y = 720（地面）
绿植高度 150-180px
绿植顶部 y = 540-570（在柜子中部，不挡柜内陈列品）

x 位置：
- 左盆：柜与椅之间的地板空间中心 → x = 540
- 右盆：柜与椅之间的地板空间中心 → x = 1060
```

### 4.3 绿植槽位

```typescript
interface PlantSlot {
  position: 'left' | 'right';   // 桌左 / 桌右
  current_plant_id: string;     // 当前展示的绿植 ID
}
```

每个槽位只展示一盆。已购买的绿植可在"装饰商店 → 绿植 → 我的绿植"里切换显示。

### 4.4 默认绿植（开局免费提供）

| 槽位 | ID | 名称 | 形态 |
|---|---|---|---|
| 左 | `plant_pothos` | 大叶绿萝 | 垂蔓 + 心形叶 |
| 右 | `plant_monstera` | 龟背竹 | 大叶 + 网状孔 |

这两盆**默认拥有**，不需积分购买。让玩家一开始书房就"有生命"。

### 4.5 SVG 绘制规范

每盆绿植：
- 整体高 150-180px
- 盆部 60×40px（陶土色或彩色）
- 叶/枝从盆口长出
- 风格：保持肥仔产品的圆润 + 手绘感
- 颜色：低饱和绿（不要荧光绿）

详细绘制参考：

```svg
<!-- plant_pothos.svg · 大叶绿萝 -->
<svg viewBox="0 0 80 180">
  <!-- Pot -->
  <path d="M15 130 L65 130 L60 170 L20 170 Z" fill="#B45F41"/>
  <ellipse cx="40" cy="130" rx="25" ry="6" fill="#823018"/>

  <!-- Leaves (heart-shaped, hanging down) -->
  <path d="M40 130 C 30 100, 15 85, 10 60 ..." fill="#6BA868" />
  <!-- ... more leaves with hanging vines ... -->
</svg>
```

### 4.6 渲染规则

绿植在画面图层上的位置：**在椅子之前，柜子之后**。

```
图层顺序（从底到顶）：
  墙 → 画 → 柜子 → 绿植 → 椅子 → 肥仔 → 桌子 → 桌面摆件
```

绿植可以**轻微遮挡椅子的左右边缘**（让画面有重叠层次感）。

### 4.7 微动效

绿植叶子轻微摇曳：

```css
.plant-leaves {
  transform-origin: bottom center;
  animation: sway 4s ease-in-out infinite;
}

@keyframes sway {
  0%, 100% { transform: rotate(-1deg); }
  50%      { transform: rotate(1.5deg); }
}
```

让书房有"风的存在感"，不是死寂的画面。

## C5 · 装饰商店扩展"绿植"类目

| `plant_cactus` | 沙漠仙人掌 | 20 ⭐ | 圆柱 + 红花 |  |

| `plant_moss` | 苔玉 | 30 ⭐ | 圆球苔藓挂件 |  |
| `plant_fiddle` | 琴叶榕 | 60 ⭐ | 大圆叶 + 树形 |  |

### 5.3 数据模型扩展

在 `userCollection` 添加：

```typescript
interface UserCollection {

  plants_active: {
    slot: 'left' | 'right';

```
